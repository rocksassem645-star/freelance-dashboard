# Freelancer Dashboard — SaaS Blueprint
> Stack: React + Tailwind + Node.js + Express + MySQL + JWT + PDFKit
> Architecture: Onion Architecture
> Hosting: Vercel (frontend) + Railway (backend + database)
> Cost: $0

---

## The Idea
A tool for freelancers to manage clients, projects, and invoices professionally.
Solves the problem of scattered tracking and late payments in Egypt and the UK.

---

## Database Tables

### users
```sql
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  country VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### clients
```sql
CREATE TABLE clients (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255),
  country VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### projects
```sql
CREATE TABLE projects (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  client_id INT NOT NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  status VARCHAR(50) DEFAULT 'in_progress',
  price DECIMAL(10,2),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### invoices
```sql
CREATE TABLE invoices (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  project_id INT NOT NULL,
  amount DECIMAL(10,2) NOT NULL,
  status VARCHAR(50) DEFAULT 'unpaid',
  due_date DATE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Onion Architecture — Plain English

Think of it as 4 circles inside each other:

```
[ Routes ] — HTTP layer, receives requests
  [ Infrastructure ] — talks to database and external tools
    [ Application ] — business logic, what the app actually does
      [ Core ] — pure data shapes, knows nothing about the outside world
```

### The Golden Rule:
Inner layers NEVER know about outer layers.
- Core doesn't know about MySQL
- Core doesn't know about Express
- Application doesn't know about HTTP
- Only Routes know about HTTP requests

### STAGE 1 — Server Setup
**Goal:** Get a basic Express server running with database connection.

Files to create:
- `server/index.js` — main entry point
- `server/src/infrastructure/database/db.js` — MySQL connection using env variables

What it does:

- Connects to MySQL database
- Returns "Server is working!" on GET /

---

### STAGE 2 — Authentication (JWT)
**Goal:** Users can register and login. JWT token returned on login.

Files to create:
- `server/src/core/entities/User.js` — user data shape
- `server/src/infrastructure/database/UserRepository.js` — save/find users in DB
- `server/src/application/useCases/auth/RegisterUser.js` — hash password, save user
- `server/src/application/useCases/auth/LoginUser.js` — verify password, return JWT
- `server/src/routes/authRoutes.js` — POST /register, POST /login

Key concepts:
- bcrypt for password hashing
- jsonwebtoken for JWT generation
- Middleware to protect routes (verify token on every request)

## JWT Middleware (protect all routes)
Every route except /register and /login needs this middleware:

```javascript
const jwt = require('jsonwebtoken')

const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1]
  if (!token) return res.status(401).send('No token provided')
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET)
    req.userId = decoded.userId
    next()
  } catch (err) {
    res.status(401).send('Invalid token')
  }
}

module.exports = authMiddleware
```
